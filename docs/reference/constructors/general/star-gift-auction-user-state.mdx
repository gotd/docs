# `starGiftAuctionUserState`

Contains information about the current user's state in an auction ». The bid_amount, bid_date, bid_peer and min_bid_amount flags of starGiftAuctionUserState will all be set if the user placed a bid in the auction.

```go
type tg.StarGiftAuctionUserState struct { /* ... */ }
```

## Fields

| Name | Type | Required | Description |
| ---- | ---- | -------- | ----------- |
| `Returned` | `bool` | — | If set, the placed bid was returned to the user, because it was outbid so much that it fell out of the top starGiftAuctionState.gifts_left positions, meaning that even if no new bids are placed, the user will never receive any gifts, so the bid was completely removed from the auction, and in order to participate again the user must manually make a new bid. |
| `BidAmount` | `int64` | — | Contains the amount of the placed bid in Telegram Stars. |
| `BidDate` | `int` | — | Contains a UNIX timestamp, indicating when the bid was placed. |
| `MinBidAmount` | `int64` | — | Contains the minumum allowed bid amount in Telegram Stars, if set overrides starGiftAuctionState.min_bid_amount for the current user. |
| `BidPeer` | [`PeerClass`](/docs/reference/types/general/peer) | — | Contains the peer that will receive the gift, if you end up winning this round |
| `AcquiredCount` | `int` | yes | Contains the number of gifts that were purchased so far in the auction by the current user. |
